# Code architecture

Browser-based geography quiz where players locate 361 NCAA Division I schools
on an Albers-projected US map. The game is written in strict ESM TypeScript,
bundled by esbuild, and served as a static site from `dist/` (GitHub Pages).
Data is generated by Python scripts from CSV files in `data/`.

## Major components

### src/ TypeScript modules

All game logic lives in `src/`. Modules follow a strict dependency order: leaf
utilities first, then state, then rendering, then wiring.

| Module | Role | Key exports |
| --- | --- | --- |
| [src/types.ts](../src/types.ts) | Shared interfaces and string-union types used by every module. | `NCAASchool`, `GameState`, `GameResults`, `AnswerRecord`, `DifficultyTier`, `StatePathData`, `Subdivision`, `TierType`, `GameScreen` |
| [src/dom_utils.ts](../src/dom_utils.ts) | Typed DOM element lookup helpers. | `getElement<T>` (throws on missing), `findElement` (returns null) |
| [src/map_projection.ts](../src/map_projection.ts) | Albers Equal-Area Conic projection for the continental US. Ports the original `parts/map_projection.js`. | `albersProjection(lon, lat)` |
| [src/data/schools_data.ts](../src/data/schools_data.ts) | Generated typed ESM module from `build_school_data.py`. | `NCAA_SCHOOLS`, `DIFFICULTY_TIERS`, `SUBREGIONS` |
| [src/data/map_paths_data.ts](../src/data/map_paths_data.ts) | Generated typed ESM module from `generate_map_paths.py`. | `US_STATE_PATHS` |
| [src/data_loader.ts](../src/data_loader.ts) | Central re-export hub for `src/data/`. Other modules import from here rather than from `src/data/` directly. | Re-exports `NCAA_SCHOOLS`, `DIFFICULTY_TIERS`, `SUBREGIONS`, `US_STATE_PATHS` |
| [src/constants.ts](../src/constants.ts) | Game constants and tier-filter helpers. Re-exports `SUBREGIONS` from `data_loader`. | `getSchoolsForTier(tierName)`, `countSchoolsForTier(tierName)`, `SUBREGIONS` |
| [src/game_state.ts](../src/game_state.ts) | Mutable game state singleton and all state-transition functions. Does not touch the DOM. | `gameState`, `startGame()`, `recordAttempt()`, `scoreAnswer()`, `advanceAfterReveal()`, `getResults()`, `saveBestScore()`, `loadBestScore()`, `getElapsedFormatted()` |
| [src/game_ui.ts](../src/game_ui.ts) | All DOM and SVG rendering: screen transitions, map dots, zoom circle, sidebar, state labels, score display, and share text. Owns module-local UI state exported as live bindings. | `renderMap()`, `showScreen()`, `showSetupScreen()`, `showResultsScreen()`, `initSidebar()`, `renderStateLabels()`, `updateCursorLabel()`, `copyShareResults()`, plus dot/highlight helpers |
| [src/game_play.ts](../src/game_play.ts) | Event handlers and the per-question loop: processes dot clicks, scores answers, triggers reveal feedback, and advances to the next question. | `showNextQuestion()`, `handleDotClick()`, `handleNextButton()`, `startTimerUpdates()`, `stopTimerUpdates()` |
| [src/init.ts](../src/init.ts) | App bootstrap and DOM event wiring. Queries all DOM elements, attaches listeners for start, next, share, play-again, theme toggle, state-labels checkbox, and keyboard shortcuts, then shows the setup screen. | `initApp()` |
| [src/main.ts](../src/main.ts) | Thin entry point. Registers a single `DOMContentLoaded` listener that calls `initApp()`. | (entry only) |

### Module dependency order

```text
types.ts  dom_utils.ts  map_projection.ts
  |           |              |
  +-----+-----+--------------+
        |
   data_loader.ts (re-exports src/data/*.ts)
        |
   constants.ts
        |
   game_state.ts
        |
   game_ui.ts <-- game_play.ts
        |
   init.ts
        |
   main.ts  (DOMContentLoaded entry)
```

### Data generators (Python)

| Script | Input | Output |
| --- | --- | --- |
| [build_school_data.py](../build_school_data.py) | `data/ncaa_schools-*.csv`, caches (`school_coordinates_cache.json`, `school_colors_cache.json`) | `src/data/schools_data.ts`, `data/schools.json` |
| [generate_map_paths.py](../generate_map_paths.py) | `data/map_paths.json` | `src/data/map_paths_data.ts` |
| [generate_debug_map.py](../generate_debug_map.py) | `data/schools.json`, `data/map_paths.json` | `debug_map_*.html` (repo root, not committed) |

### Static assets

- [src/index.html](../src/index.html) - Game markup; merged from the legacy three-fragment `parts/*.html`.
  Loads `dist/main.js` as an ES module.
- [src/style.css](../src/style.css) - Game styles; moved from `parts/style.css`.

### Build and serve scripts

- [build_github_pages.sh](../build_github_pages.sh) - Canonical production build.
  Runs `tsc --noEmit` for type-checking, then `esbuild` to bundle `src/main.ts`
  into `dist/main.js` (ESM, es2020, minified, with sourcemap). Copies
  `src/index.html` and `src/style.css` into `dist/`.
- [run_web_server.sh](../run_web_server.sh) - Local preview. Runs
  `build_github_pages.sh`, then serves `dist/` on a random port via
  `python3 -m http.server`.

allow parity checks between data regenerations.

## Extension points

- **New schools or tiers**: edit the CSV files in `data/`, then run
  `build_school_data.py` to regenerate `src/data/schools_data.ts`.
- **New game screens**: add a `GameScreen` value to `src/types.ts`, handle it
  in `game_state.ts` transitions, and render it in `game_ui.ts`.
- **New UI interactions**: add event listeners in `src/init.ts` and handlers
  in `src/game_play.ts` or `src/game_ui.ts`.
- **Map geometry changes**: update `data/map_paths.json` (hand-curated),
  then run `generate_map_paths.py` to regenerate `src/data/map_paths_data.ts`.

## Known gaps

- `src/index.html` is not separately documented; its structure (screen divs,
  SVG map container, sidebar) should be reviewed if major DOM changes are made.
- The `generate_debug_map.py` output files (`debug_map_*.html`) at the repo
  root are untracked; whether they should be moved under `baseline/debug_map/`
  or kept at the root is an open question.
